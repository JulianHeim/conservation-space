Sirma Platform : Save iDoc

An iDoc may be saved when a new iDoc is created for the first time or after editing an existing iDoc. (see also topics: "Create New iDoc" and "Edit iDoc")

  • To save the changes that have been made to the document content the user have to select  the "Save" action.
    • When a new iDoc is created, and saved for first time, the following properties in the document Properties section are set automatically:
      • Unique ID - the unique document identifier in the system
      • Title - If the user does not change the title form the "Properties" section, the title of the document is set automatically to the name of the template.
      • Created from - the user who created the document
      • Created on - the date and time the document has been created
      • Version - the version of the document is set to 1.0
All changes in the document content are also saved.

    • The user could edit and save a document many times during its creation. When the user selects the "Save" action for a document then:

      • All the changes made in the document content (widgets, sections, text, formatting) are saved.
      • In the document properties section the following fields are updated:
        • Last Modified By - the user who made the latest changes
        • Last Modified on - the date and time the changes in the document have been saved
        • Version - Saving changes in a document creates always a new minor version of the document (1.1, 1.2, etc.)
      • The Edit mode of the document changes to Preview mode.
      • A new record is created in Versions section of the document - the previous version of the document is saved and displayed in the Version section so any user could always review and revert to a previous version if necessary. (For example the newest version is 0.4, the previous version is 0.3 and it is displayed in the Versions section, where we already have records for 0.1 and 0.2 versions.) For more details on Versions section of a document see iDoc Content Management.
